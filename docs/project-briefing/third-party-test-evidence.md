# 第三方测试记录证据索引

本文用于项目展示和项目定位校准，依据外部第三方功能测试记录整理。测试原件不复制进仓库；公开版只记录可讲解的页码级证据、测试项和项目支撑边界。

## 文件摘要

- 项目名：许可链生态用户身份与数据监管系统项目第三方测试项目。
- 机构：第一页显示为中国赛宝实验室。
- 页数：26 页。
- 主要执行时间：2024.10.15 到 2024.10.18。
- 相关系统：`GS_TBK_version2_9_1`、FISCO BCOS（金融区块链合作联盟开源区块链底层平台）、console（控制台）、id_info（身份字段处理模块）。

## 不建议作为本项目主线展开的内容

PDF 后半部分还包含跨链密文监管、账户-交易-链群知识库、交易欺诈风险监测、敏感词检测和链上异常数据重编辑等测试项。这些内容可以作为同一测试记录集里的背景，但当前仓库主线主要是分布式身份监管、身份字段密文上链、签名上链和异常揭示，不建议在本项目项目材料中大篇幅展开：

| 页码 | 测试项 | 处理建议 |
| --- | --- | --- |
| 第 14-16 页 | 用户隐私数据密文跨链监管 | 除非投跨链方向，否则不作为本项目核心贡献 |
| 第 17-19 页 | 许可链账户-交易-链群知识库 | 可作为数据监测背景，不写进核心技术栈 |
| 第 20-21 页 | 用户链上交易欺诈风险监测 | 涉及机器学习分类，但不是当前 GS-TBK（Group Signatures with Time-bound Keys，带时间绑定密钥的群签名方案）项目主线 |
| 第 22-23 页 | 用户链上数据敏感词检测 | 可作为数据安全背景，不要包装成完整 AI（Artificial Intelligence，人工智能）安全系统 |
| 第 24-26 页 | 链上异常数据重编辑 | 与当前仓库主线差异较大，讲解中谨慎展开 |

## 推荐项目材料版本

```text
项目概述：参与研发一套面向许可链的分布式身份监管系统。系统基于 GS-TBK 群签名方案与 FISCO BCOS 区块链，通过委员会节点的阈值密钥生成、用户匿名签名、链上身份数据登记和异常身份揭示机制，在保护用户隐私的前提下实现恶意用户追责。项目在 5 台云服务器和 1 台本地服务器环境中完成第三方功能测试用例验证，覆盖阈值密钥生成、用户注册、签名上链、身份揭示、隐私数据密文上链和解密校验等流程，相关测试结论均为通过。

核心技术栈：Rust（系统级编程语言）、FISCO BCOS、GS-TBK 群签名、CL 同态加密、ZKP 零知识证明、Solidity（智能合约编程语言）、Java SDK（Software Development Kit，软件开发工具包）

个人核心贡献：
• 身份字段处理与 CL 同态加密模块实现：负责用户身份字段的数据处理、编码拼接、CL 同态加密、承诺和零知识证明生成流程，编写 Rust 程序调用加密库，将身份密文、承诺和证明数据通过智能合约写入 FISCO BCOS，并实现监管侧链上数据读取、解密验证和字段还原。

• 系统集成测试与第三方测试支撑：负责多节点部署联调和测试记录整理，在 4 个委员会节点、1 个代理节点和本地用户节点环境中跟踪 KeyGen（密钥生成）、用户注册、签名上链、链上查询、Verify/Open 身份揭示等流程；通过日志和链上数据核验异常用户揭示结果，支撑第三方功能测试用例通过。
```

## 口径边界

- 可以说“第三方功能测试用例验证通过”，但如果没有最终验收报告，不建议直接说“完整验收通过”。
- 可以说“5 台云服务器和 1 台本地服务器测试环境”，因为 PDF 多处写明该拓扑。
- 可以说“异常数据拥有者身份揭示”，第 9-10 页能支撑；讲解中再结合当前仓库 2026-05 真实 VM（Virtual Machine，虚拟机）复现证据说明 Verify/Open（验证/揭示）细节。
- 不建议继续写 BBS+（一种选择性披露签名方案），因为当前仓库和第三方测试记录都不能直接支撑它是主线实现。
- 不建议把第 20-23 页的欺诈检测、敏感词检测包装成当前项目已经实现的完整 AI 安全系统。

## 讲解时的讲法

这份第三方测试记录证明早期项目确实在多服务器环境中跑过功能测试，覆盖阈值密钥生成、注册、签名上链、异常身份揭示和隐私数据密文上链。当前仓库的价值是把这些历史材料进一步整理成可复现工程原型，并在 2026-05 重新完成真实 VM 上的 E2E（End-to-End，端到端）smoke（冒烟验证）、链上审计查询和恶意揭示摘要。因此讲解时建议把两类证据分开讲：第三方测试记录证明项目来源和功能范围，当前仓库证明我能复现、工程化整理和演示闭环。
