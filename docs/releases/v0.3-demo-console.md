# v0.3 演示控制台发布说明

本文是 `v0.3-demo-console` 的 release（发布）说明，面向项目演示、项目讲解和后续维护。它基于 v0.2 可审计工程原型继续补齐只读 Web（网页）/API（Application Programming Interface，应用程序接口）审计控制台、event schema（事件模式）和 AI（Artificial Intelligence，人工智能）安全桥接材料。

## 当前定位

当前项目可以定位为：

> 可展示、可审计、可项目讲解的分布式身份监管工程原型。

v0.3 的重点不是扩大协议本身，而是把 v0.2 已经沉淀的真实 VM（Virtual Machine，虚拟机）E2E（End-to-End，端到端）证据、链上审计查询、恶意用户 Verify/Open（验证/揭示）和 JSON（JavaScript Object Notation，数据交换格式）摘要，整理成一个可被浏览器和 API 只读消费的演示入口。

它仍然不是可上线生产系统。`audit-console`（审计控制台）不连接真实 FISCO BCOS（金融区块链合作联盟开源区块链底层平台）链节点，不读取真实证书、私钥、wallet（钱包）、keystore（密钥库）、`conf/config.toml` 或运行大日志，也不替代权限控制、长期监控、告警、灾难恢复和生产级审计平台。

## v0.3 新增能力

- 只读 `audit-console` 演示层已纳入正式里程碑：它聚合 `examples/evidence/*.json` 和 `docs/evidence/*.json`，通过 Web 页面与 `/api/evidence` 展示 User（用户）、Contract（合约）、TX（Transaction，交易）、区块、查询判读、Verify/Open 状态、揭示字段和失败场景。
- API 展示边界已明确：`GET /api/health` 用于健康检查，`GET /api/evidence` 返回聚合后的只读证据模型；服务不访问真实链端配置、证书、私钥或大日志。
- `events[]` 事件样例成为演示数据入口：`examples/evidence/events.sample.json` 覆盖 `audit_query`、`malicious_open` 和 `failure_scenario` 三类事件，便于把链上审计查询、恶意揭示和失败场景统一展示。
- `event-schema.md` 已定义结构化事件模式：每个事件包含 `event_id`、`event_type`、`timestamp`、`source`、`actor`、`target`、`chain`、`status`、`evidence_refs`、`risk_level` 和 `summary` 等字段。
- AI 安全桥接材料已进入 release 口径：当前项目不是大模型训练或 Prompt Injection（提示词注入）防护系统，但它可以作为 AI 安全场景中身份可信、行为留痕、异常追责和多方监管的工程底座讲解。

## 最短演示路径

以下命令在仓库根目录执行，需要 Node（JavaScript 运行时）18 或更新版本。

```bash
cd apps/audit-console
npm run check
npm start
```

默认访问地址：

```text
http://127.0.0.1:4173
```

`npm run check` 只验证 JSON 摘要读取边界和聚合结构，不启动 Web 服务。期望结果中 `ok` 为 `true`，且读取来源只包含 `examples/evidence/*.json` 和 `docs/evidence/*.json`。

## 演示讲解路径

1. 先打开 `http://127.0.0.1:4173`，说明这是只读展示层，只消费已提交的证据摘要。
2. 展示链上审计查询：用户记录可追溯到合约、TX 哈希和区块高度，并能区分当前查询、登记区块历史查询和登记前一区块查询。
3. 展示恶意揭示事件：异常签名 Verify 失败后触发 Open，并在摘要中保留 `user_id`、`user_name` 和地址等揭示字段。
4. 展示 `events[]`：说明 `audit_query`、`malicious_open` 和 `failure_scenario` 被统一抽象成机器可读事件，后续可服务 Web/API、演示报告和 AI 安全审计材料。
5. 收束到 AI 安全桥接：把“匿名使用、行为留痕、异常追责、多方监管”迁移到智能体工具调用审计、模型服务敏感操作追责或数据访问留痕，但不宣称已经实现大模型安全平台。

## 安全边界

- 不连接真实 FISCO BCOS 节点，不发链上交易，不查询运行中的生产链。
- 不读取、不展示、不提交真实证书、私钥、账户文件、wallet、keystore、`conf/config.toml`、runtime-state（运行时状态）或运行大日志。
- 控制台只读取已经入库的 JSON 摘要和脱敏 fixture（夹具）事件样例。
- `events[]` 是现有证据摘要的旁路聚合层，不替换原始 Markdown（轻量标记语言）报告、JSON 摘要、manifest（运行清单）或日志 SHA-256。
- AI 安全桥接是能力迁移说明，不代表当前仓库已经具备大模型训练、模型安全评测、提示词注入防护或生产级 AI 治理能力。

## 验收 checklist

- `apps/audit-console/README.md` 能说明只读 Web/API 展示范围和安全边界。
- `npm run check` 在 `apps/audit-console` 下通过，并且只扫描 `examples/evidence/*.json` 与 `docs/evidence/*.json`。
- `npm start` 后可访问 `http://127.0.0.1:4173`。
- `/api/evidence` 返回 `sources`、`users`、`contracts`、`transactions`、`query_results`、`verify_open`、`reveal_fields`、`failure_scenarios` 和 `events` 等只读聚合字段。
- `events[]` 样例覆盖 `audit_query`、`malicious_open` 和 `failure_scenario`，并能回指证据来源。
- 发布说明明确 v0.3 是“可展示、可审计、可项目讲解”的工程原型，不夸大为可上线生产系统。

## 已知限制

- 当前控制台是演示用只读展示层，不是带认证授权、租户隔离、实时采集、权限审计和告警能力的生产平台。
- `/api/evidence` 聚合的是已提交摘要和样例事件，不保证覆盖每一次真实 VM 运行。
- 结构化 `events[]` 目前以样例和 schema 为主，后续如需生产生成器，应从现有摘要只读派生并保留 `evidence_refs`。
- AI 安全桥接目前是文档与讲解口径，后续若要形成毕设 demo（演示工程），还需要新增模型服务、工具调用事件或训练审计合约。
